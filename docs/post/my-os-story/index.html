My Story with OS's

This my story with OS's
2024-01-21 · 2 min · 399 words · Me

My journey with desktop OS along years#

As the majority my most used desktop OS until now is Windows 10 (I had touch XP, 7, 8, 10, 11). However in the year 2022 I switch to Linux (version 6.0) on my main machine looking for a more private, secure and open-source system running away from Windows. I have used “Linux desktop” before on one of my laptops but I didn’t get it.

So when I decided to change my desktop OS to “Linux desktop”, I started “the hard way” choosing Arch Linux, because once you boot your OS need to install manually the system typing commands. I remember watching many YouTube videos of the manual installation, reading the installation guide on the Arch wiki and trying the Arch installation on virtualbox on Windows.

But it was so hard for me I didn’t dare to do it, however I discovered Arco Linux (from Erik Dübois) an Arch-based Linux distro with a GUI installer, just clicking buttons and selecting options. Also in that time I started classifiyng all the Linux distros, desktop environments and compatible software doing that on my Interneto project. I stayed with Arco for a few months (in the middle I broke the system with archlinux-tweak-tool and need to reinstall) but I didn’t like the appareance so much and I changed to XeroLinux. With XeroLinux configured I stayed another few months and after “Dolphin” (KDE file manager) corrupting one of my external HDD I ended up switching all to “Vanilla” (no extra configuration) with Arch + KDE. Then learning more the way that Arch Linux works to finally create my own Arch-based Linux distro ArkSys.

But after all of that I have discovered that I don’t need a GUI installer for a fresh install because I can use archinstall (an easy way selecting options throw the terminal) and just import config files. However sometimes a configured system with a GUi installer is necessary, for example using an OS throw a USB without installing.

So my first year of Linux was Distro-hoping, which means switching between different distros looking for your “perfect Linux distro” even real or virtual machine. And until today, these are all the distos I have installed on a VM:

  • Arch-based: ArcoLinux, XeroLinux, Garuda Linux, CachyOs, Archcraft, EndeavourOS
  • Debian-based: Ubuntu, Kubuntu, Linux Mint, kali Linux, ParrotOS, Pop-OS, ProxMox
  • Fedora-based: Fedora, Nobara
  • OpenSuse-based: OpenSuse
  • Indie: Void Linux
